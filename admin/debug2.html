Effective Settings Snapshot POST action=get_settings

Loads env values, KV overrides, and the final effective settings used by the site.

KV Peek GET ?type=debug_kv_peek

Fetches and dumps a single KV key as JSON for quick inspection.

Choosing a preset will fill the key field below. Requires a debug handler like ?type=debug_kv_peek&key=....

Scheduler / Cron Triggers POST actions

Manually run the same logic used by cron jobs, using your current admin token.

Uses action=send_monthly_chair_reports and action=send_end_of_event_reports in /api/router.

Mail Log History GET ?type=debug_mail_recent

Returns the most recent N entries from the mail log (chair reports, contact forms, test messages, etc.).

Requires a handler like ?type=debug_mail_recent&limit=20.

Order Search GET ?type=debug_find_orders

Finds recent orders by purchaser, attendee, email, or partial ID. Returns a small list with IDs and basic info.

Requires ?type=debug_find_orders&q=...&limit=....

Chair Routing Snapshot GET ?type=debug_chair_routing

Shows each registered item’s chair emails, publish window, and reportFrequency, so you can see where reports are going.

Can be implemented either as a new debug_chair_routing type or by reusing debug_itemcfg_health.

Contact Form Simulator POST action=contact_form

Sends a test message through the normal contact form pipeline using a clearly marked debug payload.

Uses action=contact_form with a page marker of debug2.

Stripe PaymentIntent Preview GET ?type=debug_pi_preview

Fetches a PaymentIntent by ID and shows amount, status, charges, etc.

Requires a handler like ?type=debug_pi_preview&pi_id=....

Order + Refunds Preview GET ?type=debug_order_refunds

Shows an order plus any refunds recorded against its payments.

Can be implemented separately, or re-use your existing debug_order_preview and include refund rows.
